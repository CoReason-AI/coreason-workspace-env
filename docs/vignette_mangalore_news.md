# Vignette: Building a Mangalore News Agent

This vignette demonstrates how a single user can start the `coreason-workspace-env` and collaborate with the **Factory CEO** (`factory_ceo`) to design, build, validate, and publish a simple agentic application. Our goal is to create an agent that discusses the current local news in Mangalore, India.

## 1. Starting the Workspace

First, ensure your `.env` file is configured with the necessary LLM and infrastructure credentials. Then, start the workspace in standalone mode using Docker Compose:

```bash
docker-compose -f docker-compose.yaml -f docker-compose.standalone.yaml up -d
```

This spins up the complete headless agent development platform, including the Postgres checkpointer, MinIO WORM storage, Vault, and the main platform server.

## 2. Engaging the Factory CEO

The platform offers multiple interaction surfaces (CLI, REST API, MCP). For this vignette, we'll use the CLI's `build` command to start an engagement with the top-level orchestrator, the `factory_ceo`.

```bash
uv run coreason build "I want to create a simple agentic application that discusses the current local news in Mangalore, India."
```

## 3. The Orchestrator Interrogation Loop

Because the `factory_ceo` operates as a **State Machine Orchestrator** (adhering to the Context Engineering Harness philosophy), it doesn't immediately generate code. It realizes the context is underspecified and enters an interactive loop to saturate the context payload. The CLI will pause and prompt you:

> **[CEO]** I can help you build this agent. To ensure we build exactly what you need, please clarify:
> 1. Should this agent actively scrape specific local news sources (e.g., Daijiworld, Udayavani), or rely on general web search?
> 2. What is the intended output format? (e.g., a daily markdown summary, an interactive Q&A bot)
> 3. Do we need any specific MCP integrations for deployment?
> **You:** It should use general web search via Tavily. The output should be an interactive Q&A bot. We want to publish it as an MCP server.

## 4. Delegation and the Builder-Validator-Approver Workflow

With a fully saturated context, the `factory_ceo` stops talking to the user and instantly delegates the raw context payload to its deterministic sub-agents. 

1. **Planning (`agent_pm`):** Creates the project structure and tracking checklist.
2. **Building (`prompt_engineer` & `yaml_compiler`):** 
   - The `prompt_engineer` writes the system prompt detailing how to summarize and discuss Mangalore news.
   - The `yaml_compiler` writes the deterministic `agent.yaml` definition (in `src/agents/mangalore_news/agent.yaml`), wiring up the Tavily search tool and the LangGraph routing logic.
3. **Validation (`agent_validator`):** The validator checks the generated artifacts against the `src/core/skills/validation/` standards to ensure the schema is pure, the ID matches the folder namespace (`mangalore_news`), and the workflow is decoupled.

*You can observe this entire process in real-time via the CLI streaming output or a WebSocket streaming "Accordion" UX.*

## 5. Publishing the Agent

Once the validation is complete, the application is bundled into a standalone deployable package. The CLI will output:
`[SUCCESS] Platform bundled at: C:\...\dist\<session-id>.zip`

This ZIP file contains the full `orchestrator_agent.yaml`, `project.yaml`, and `pyproject.toml` generated by the factory. 

Because of the platform's multi-surface interaction parity and OCI-native design, you can now push this bundled project to an upstream registry using the CLI. By default, it is recommended to push to a local testing registry:

```bash
uv run coreason push-project <your-project-id> localhost:5000/mangalore-news:latest --skip-state --skip-docker
```

### Alternative Registries
You can also push your agent to public or private cloud registries by swapping out the destination URL:

- **GitHub Container Registry (GHCR):**
  ```bash
  uv run coreason push-project <your-project-id> ghcr.io/<your-github-username>/mangalore-news:latest --skip-state --skip-docker
  ```
- **Docker Hub:**
  ```bash
  uv run coreason push-project <your-project-id> docker.io/<your-docker-username>/mangalore-news:latest --skip-state --skip-docker
  ```

Your Mangalore News agent is now packaged, portable, and ready to discuss the latest updates from the coastal city anywhere!
